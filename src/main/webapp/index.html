Jakarta EE 11 Application

Welcome to the Jakarta EE 11 application with JPA persistence!

Available REST Endpoints

GET /api/hello

Create a casual greeting and save it to the database

curl "http://localhost:8080/jakarta-ee-app/api/hello?name=John"
GET /api/hello/formal

Create a formal greeting and save it to the database

curl "http://localhost:8080/jakarta-ee-app/api/hello/formal?name=Jane"
GET /api/hello/greetings

Get all greetings from the database

curl "http://localhost:8080/jakarta-ee-app/api/hello/greetings"
GET /api/hello/greetings/by-name

Get greetings by specific name

curl "http://localhost:8080/jakarta-ee-app/api/hello/greetings/by-name?name=John"
GET /api/hello/stats

Get greeting statistics

curl "http://localhost:8080/jakarta-ee-app/api/hello/stats"
GET /api/hello/count

Get greeting count for a specific name

curl "http://localhost:8080/jakarta-ee-app/api/hello/count?name=John"
GET /api/hello/health

Health check endpoint

curl "http://localhost:8080/jakarta-ee-app/api/hello/health"

Database Configuration

The application uses Jakarta Persistence (JPA) with EclipseLink as the provider.

By default, it's configured to use GlassFish's default datasource, but you can configure it to use any database.

For testing, an H2 in-memory database configuration is also provided.

Features Demonstrated

  • Jakarta EE 11 / Java 21
  • JAX-RS REST endpoints
  • CDI (Contexts and Dependency Injection)
  • JPA (Jakarta Persistence) with EclipseLink
  • Bean Validation
  • JTA (Jakarta Transaction API)
  • Unit testing with JUnit 5 and Mockito
  • Integration testing with Arquillian
